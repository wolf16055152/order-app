# 주문하기 화면 PRD

> 상위 문서: [PRD.md](./PRD.md)  
> 화면 ID: `order`  
> 앱 브랜드명: **COZY**

## 1. 개요

### 1.1 목적
고객이 메뉴를 탐색하고, 옵션을 선택해 장바구니에 담은 뒤, 한 번에 주문을 접수할 수 있는 화면을 제공한다.

### 1.2 범위
- 본 문서는 **주문하기** 화면의 프런트엔드 UI·동작만 정의한다.
- **관리자** 화면, 인증, 결제, 배송 등은 별도 PRD에서 다룬다.
- 주문 접수 후 서버·관리자 쪽 처리 흐름은 최소한으로만 언급한다.

### 1.3 대상 사용자
- 매장에서 메뉴를 주문하는 일반 고객(키오스크·태블릿·PC 웹)

### 1.4 성공 지표(학습 프로젝트 기준)
- 메뉴 선택 → 옵션 선택 → 담기 → 장바구니 확인 → 주문하기까지 끊김 없이 완료 가능
- 옵션·수량에 따른 금액이 화면에 즉시 반영됨

---

## 2. 화면 구조(와이어프레임 기준)

화면은 위에서 아래로 **헤더 → 메뉴 목록 → 장바구니** 세 영역으로 구성한다.

```
┌─────────────────────────────────────────────────────────┐
│  COZY                    [ 주문하기 ]  [ 관리자 ]        │  ← 헤더
├─────────────────────────────────────────────────────────┤
│  [메뉴 카드] [메뉴 카드] [메뉴 카드] ...                   │  ← 메뉴 영역
├─────────────────────────────────────────────────────────┤
│  장바구니                                                │
│  · 항목 목록 · 총 금액 · [ 주문하기 ]                     │  ← 장바구니 영역
└─────────────────────────────────────────────────────────┘
```

## 3. UI 구성 요소

### 3.1 헤더(공통 네비게이션)

| 요소 | 설명 | 요구사항 |
|------|------|----------|
| 로고/앱명 | `COZY` | 좌측 정렬, 클릭 시 주문하기 화면으로 이동(동일 화면이면 무동작 가능) |
| 탭 `주문하기` | 현재 화면 | **활성** 상태: 테두리 등으로 현재 위치 표시 |
| 탭 `관리자` | 관리자 화면 | 비활성 스타일, 클릭 시 관리자 화면으로 라우팅 |

**동작**
- 두 탭 중 하나만 활성화한다.
- 화면 전환 시 주문하기 화면의 **장바구니 상태는 유지**할지 여부는 구현 단계에서 결정하되, 기본값은 **세션(또는 메모리) 내 유지**를 권장한다.

---

### 3.2 메뉴 영역

메뉴는 **카드 그리드**로 표시한다. 와이어프레임 예시: 아메리카노(ICE), 아메리카노(HOT), 카페라떼 등.

#### 3.2.1 메뉴 카드 구성

| 영역 | 필드 | 표시 예 | 비고 |
|------|------|---------|------|
| 이미지 | `imageUrl` | 플레이스홀더(X 박스) 허용 | 이미지 없을 때 기본 플레이스홀더 |
| 이름 | `name` | `아메리카노(ICE)` | 필수 |
| 가격 | `basePrice` | `4,000원` | 천 단위 콤마, `원` 접미사 |
| 설명 | `description` | `간단한 설명...` | 없으면 생략 또는 빈 문구 |
| 옵션 | `options[]` | 체크박스 목록 | 메뉴별로 다를 수 있음 |
| 액션 | `담기` 버튼 | — | 클릭 시 장바구니에 반영 |

#### 3.2.2 옵션(체크박스)

와이어프레임 기준 예시 옵션:

| 옵션명 | 추가 금액 | UI 라벨 예 |
|--------|-----------|------------|
| 샷 추가 | +500원 | `샷 추가 (+500원)` |
| 시럽 추가 | +0원 | `시럽 추가 (+0원)` |

**규칙**
- 옵션은 **0개 이상** 선택 가능(복수 선택).
- 선택하지 않은 옵션은 장바구니·주문에 포함하지 않는다.
- 옵션 추가 금액은 **단가에 합산**한다(수량 N일 때: `(기본가 + 옵션합) × N`).

**담기 버튼 클릭 시**
1. 현재 카드에서 선택된 옵션을 읽는다.
2. 동일 메뉴 + **동일 옵션 조합**이 장바구니에 있으면 **수량 +1**, 없으면 **신규 라인** 추가(수량 1).
3. 장바구니 영역을 즉시 갱신한다(총액 재계산 포함).

---

### 3.3 장바구니 영역

하단 **둥근 사각형 컨테이너** 안에 배치한다.

#### 3.3.1 레이아웃

| 구역 | 내용 |
|------|------|
| 좌측·상단 | 제목 `장바구니` |
| 본문(좌) | 담긴 항목 목록 |
| 우측 | `총 금액` 라벨 + 합계 금액 + `주문하기` 버튼 |

#### 3.3.2 장바구니 라인 항목

각 줄은 다음 정보를 표시한다.

| 표시 | 형식 예 | 계산 |
|------|---------|------|
| 상품·옵션·수량 | `아메리카노(ICE) (샷 추가) X 1` | 옵션이 없으면 괄호 생략 |
| 라인 소계 | `4,500원` | `(기본가 + 옵션 추가액) × 수량` |

**표기 규칙**
- 옵션이 여러 개면 괄호 안에 선택된 옵션명을 나열(순서는 메뉴 정의 순 또는 선택 순 — **일관되게 한 가지 규칙** 적용).
- 수량은 `X {quantity}` 형식.

#### 3.3.3 총 금액

| 요소 | 설명 |
|------|------|
| 라벨 | `총 금액` |
| 값 | 모든 라인 소계의 합, 예: `12,500원` |
| 갱신 | 담기·수량 변경·항목 삭제 시 **실시간** 반영 |

#### 3.3.4 주문하기 버튼

| 상태 | 조건 | 동작 |
|------|------|------|
| 활성 | 장바구니에 1개 이상 항목 | 주문 API 호출(또는 목업), 성공 시 장바구니 비우기 + 완료 피드백 |
| 비활성 | 장바구니 비어 있음 | 클릭 불가, 시각적으로 비활성 표시 |

**주문 성공 시(최소 UX)**
- 토스트/알림 또는 간단한 확인 메시지
- 장바구니 초기화
- (선택) 주문 번호 표시 — 백엔드 스펙 확정 후 반영

**주문 실패 시**
- 에러 메시지 표시, 장바구니 유지

## 4. 데이터·상태

### 4.1 메뉴(서버 또는 정적 목록)

```ts
// 개념 모델 (구현 언어에 맞게 조정)
MenuItem {
  id: string
  name: string
  basePrice: number      // 원 단위 정수
  description?: string
  imageUrl?: string
  options: MenuOption[]
}

MenuOption {
  id: string
  name: string           // 예: "샷 추가"
  extraPrice: number     // 0 이상
}
```

초기 시드 예(와이어프레임):

| name | basePrice | options |
|------|-----------|---------|
| 아메리카노(ICE) | 4,000 | 샷 +500, 시럽 +0 |
| 아메리카노(HOT) | 4,000 | 동일 |
| 카페라떼 | (정의 필요) | 동일 또는 메뉴별 상이 |

### 4.2 장바구니 라인

```ts
CartLine {
  menuItemId: string
  menuName: string       // 표시용 스냅샷
  selectedOptionIds: string[]  // 정렬된 ID 배열 — 동일 조합 판별용
  selectedOptionLabels: string[] // 표시용
  unitPrice: number      // basePrice + sum(option.extraPrice)
  quantity: number
  lineTotal: number      // unitPrice * quantity
}
```

**동일 항목 판별 키:** `menuItemId` + `selectedOptionIds`(정렬 후 직렬화)

### 4.3 화면 로컬 상태

| 상태 | 설명 |
|------|------|
| `menus` | 메뉴 목록(로딩/에러 포함) |
| `cartLines` | 장바구니 라인 배열 |
| `cartTotal` | 파생 값: `sum(lineTotal)` |
| `orderSubmitting` | 주문 요청 중 로딩 |

## 5. 비즈니스 규칙

1. **가격 표시**: 모든 금액은 원화 정수, UI에는 천 단위 구분 콤마.
2. **옵션 가격**: 0원 옵션도 라벨에 `(+0원)` 표기 가능(와이어프레임 일치).
3. **수량**: 담기 시 기본 +1; 동일 키 항목은 수량만 증가(별도 라인 생성 안 함).
4. **최소 주문**: 장바구니 1개 이상일 때만 주문 가능.
5. **인증·결제**: 없음([PRD.md](./PRD.md) 프로젝트 기본 사항).

## 6. API 연동(참고)

프런트엔드·백엔드 분리 개발 전제. 엔드포인트명은 구현 시 확정.

| 동작 | 메서드 | 설명 |
|------|--------|------|
| 메뉴 목록 조회 | `GET /menus` | 카드 렌더링용 |
| 주문 생성 | `POST /orders` | body: 장바구니 라인 배열(메뉴 id, 옵션 ids, 수량, 단가 스냅샷) |

주문 요청 body 예(개념):

```json
{
  "items": [
    {
      "menuItemId": "americano-ice",
      "optionIds": ["extra-shot"],
      "quantity": 1,
      "unitPrice": 4500
    }
  ],
  "totalAmount": 12500
}
```

메뉴 목록은 1차 개발에서 **정적 JSON**으로 시작해도 된다.

## 7. 예외·엣지 케이스

| 상황 | 기대 동작 |
|------|-----------|
| 메뉴 로딩 중 | 스켈레톤 또는 로딩 문구 |
| 메뉴 0건 | 빈 상태 메시지 |
| 메뉴 로드 실패 | 재시도 안내 |
| 담기 후 옵션만 변경하고 다시 담기 | **새 옵션 조합**이면 새 라인, 같으면 수량 증가 |
| 주문 중 중복 클릭 | 버튼 비활성화, 중복 제출 방지 |
| 네트워크 오류 | 에러 메시지, 장바구니 유지 |

## 8. 접근성·반응형(권장)

- 버튼·체크박스에 키보드 포커스 가능
- 활성 탭에 `aria-current="page"` 등 시맨틱 속성
- 좁은 화면: 메뉴 카드 **세로 스택** 또는 가로 스크롤(와이어프레임은 가로 배치)

## 9. 수용 기준(Acceptance Criteria)

- [ ] 헤더에 `COZY`, `주문하기`(활성), `관리자`(비활성)가 보인다.
- [ ] 메뉴 카드에 이미지(또는 플레이스홀더), 이름, 가격, 설명, 옵션 체크박스, `담기`가 있다.
- [ ] 옵션 선택에 따라 단가가 반영되고, `담기` 시 장바구니에 반영된다.
- [ ] 동일 메뉴·동일 옵션 조합 담기 시 수량이 증가한다.
- [ ] 장바구니에 `이름 (옵션) X 수량`과 라인 소계가 표시된다.
- [ ] `총 금액`이 모든 라인 합과 일치한다.
- [ ] 장바구니가 비어 있으면 `주문하기`가 비활성이다.
- [ ] `주문하기` 성공 시 피드백 후 장바구니가 비워진다.
- [ ] `관리자` 탭 클릭 시 관리자 화면으로 이동한다.

## 10. 미포함(Out of Scope)

- 로그인·회원·포인트
- 카드/현금 결제 UI
- 장바구니 항목 삭제·수량 직접 수정 UI(와이어프레임 미표기 — **2차 개선** 후보)
- 쿠폰·할인
- 다국어

## 12. 와이어프레임 참조

주문하기 화면 와이어프레임(메뉴 카드 3종, 장바구니 예시 3품목·총 12,500원)을 UI 레이아웃·카피의 기준으로 삼는다.
